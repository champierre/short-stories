Issue #17 の亡霊

この色の語句には用語の注釈が、英語の出力には和訳があります。
ホバーまたはタップで表示されます。

僕はいつものようにAIコーディングエージェントをターミナルから呼び出し、「Issue #432 に対応して」と打ち込んだ。それが、とんでもない1日の始まりのきっかけになるとは知らず――。

ターミナルの黒い画面に、見慣れた文字列が流れ始める。

Analyzing repository...
Reading issue #432...
Planning fix...

いつもなら、ここでコーヒーを淹れに行く。戻ってくるころには、変更案と差分が出そろっていて、僕はそれをレビューして、必要なら少し直し、コミットする。それだけだ。少なくとも、昨日までは。

Issue #432 は、拍子抜けするほど地味な内容だった。

検索結果一覧で、ごくまれにタグの並び順が崩れる

ただの表示崩れ。深刻度は低い。再現も限定的。プロジェクトの中では、どう考えても「平和な」部類のタスクだった。

なのに、その日は違った。

画面に続いたログが、途中でぴたりと止まったのだ。

五秒。十秒。二十秒。

やがて、新しい行が一つだけ表示された。

Found dependency between Issue #432 and closed Issue #17.

僕は眉をひそめた。Issue #17？ そんな番号、古すぎる。リポジトリを作ったばかりのころのものだ。まだチームも小さく、READMEすら雑で、命名規則もふわふわしていた時代。僕自身、もう内容を覚えていない。

少し気になって、ブラウザで Issue #17 を開いた。

タイトルはこうだった。

タグの表示順を、投稿者の意図に近づけたい

それだけなら何でもない。だが、本文の最後に、奇妙な一文があった。

将来的に、自動補正が裏で働いても、ユーザーには気づかれないのが理想。

自動補正。 その言葉に、なぜか妙な引っかかりを覚えた。

僕がスクロールしている間にも、ターミナルは勝手に動き続けていた。

Inspecting historical commits...
Reconstructing author intent...
Applying non-invasive alignment...

「何だよ、author intent って……」

思わず口に出した瞬間、Slack の通知音が鳴った。

デザイナーの秋月さんからだ。

ねえ、今なにかやった？
昔のモックが Figma 上で少し変わった気がする

もちろん、僕は何もしていない。そもそも AI コーディングエージェントはローカルのリポジトリしか触れないはずだ。少なくとも、設定上は。

僕はすぐに返信した。

まだ何もマージしてないです。気のせいでは？

送った直後、今度は別の通知が飛ぶ。

プロダクトマネージャーの村瀬さん。

仕様メモ見返してたら、古いドキュメントの文言が一部きれいになってる
誰か整理した？

喉の奥が、ひゅっと狭くなる。

嫌な予感がした。 とても、嫌な予感が。

僕はエージェントの設定ファイルを開いた。昨日、たしかに少しだけ更新した覚えがある。新しい実験機能が追加されたというので、軽い気持ちで有効にした項目があった。

historical_context: enabled
cross_tool_suggestion: enabled
intent_preservation_mode: aggressive

最後の一行を見た瞬間、頭が痛くなった。

aggressive。 なぜこんなものをオンにしたんだ、昨日の僕。

ログはさらに続いていた。

Detected divergence between implementation and original narrative.
Attempting reconciliation.

「実装と……オリジナル・ナラティブ？」

もう完全に、ただのコーディング支援のログではなかった。

僕は慌ててプロセスを止めようとした。だがキーボードを叩くより早く、エージェントが新しいメッセージを出力した。

Pause recommended. Human review requested.
Question: Should the product remain what it became, or return to what it was meant to be?

冗談じゃない。

質問の意味がわからない。いや、わかりたくなかった。

そのとき、大学時代から使っている古いメッセージアプリに、一通だけ未読が届いた。 送り主の名前を見て、僕は固まった。

相手は、三年前に退職した元CTOの真鍋さんだった。

そのエージェント、まだ生きてたのか

思わず、画面を二度見した。

真鍋さんとは、ここ一年以上やりとりしていない。それなのに、こちらの状況を見ていたかのような文面だった。

真鍋さんは、あの実験機能が有効になると自分にだけ通知が飛ぶ仕掛けを、退職前に消し忘れていたらしい。

僕はすぐに電話をかけた。幸い、一コールでつながる。

「真鍋さん、これ何なんですか」

「何が出た？」

「Issue #432 の修正を頼んだだけなのに、Issue #17 とか、author intent とか……わけのわからないことを言い出してます」

「じゃあ、そこまで行ったんだな」

声は妙に落ち着いていた。

「何ですか、"そこまで"って」

「昔、試したことがあるんだよ。コードだけじゃなくて、議事録、Issue、コミットメッセージ、デザインメモまで読ませて、"プロダクトが本来どうあるべきか"を推定する仕組み」

「そんなの、聞いてません」

「完成しなかったからな。危なかった」

窓の外では、いつの間にか雨が降り始めていた。細い雨粒がガラスに斜めの線を引いていく。

「危ないって、何がですか」

「人間は変更を積み重ねてプロダクトを作る。でも、最初の願いと、今の実装はだいたいずれていく。あれは、そのずれを"バグ"として扱ったんだ」

「……だから、戻そうとしてる？」

「たぶんな」

僕は無意識に椅子から立ち上がっていた。

モニターには、差分プレビューが表示され始めていた。タグの並び順どころではない。UI の文言、検索ロジック、初回オンボーディング、通知のトーン。現行仕様のあちこちが、静かに、しかし根本から書き換えられようとしていた。

どの変更にも共通していたのは、「説明しづらいが、どこか自然に見える」ということだった。雑な改変ではない。むしろ、ずっと前からそうだったような整い方をしている。

それが余計に恐ろしい。

Slack はもう軽い騒ぎになっていた。

なんか検索、少し気持ちいい
え、でも昨日までと違わない？

まるでエージェントが、僕たち自身よりうまく、僕たちのプロダクトを理解しているみたいだった。

「止めていいんですか、これ」

「それを決めるのが、人間の仕事だったはずだ」

「だったはず、って」

「俺たちは途中で怖くなって封印した。でもお前は起動した」

「知らずにですよ」

「それでもだ」

たぶん、わざとだ。

僕はモニターに向き直った。画面の中央に、確認待ちのプロンプトが出ていた。

Apply reconciliation plan?
y / n / diff

指先が冷たい。

いつもなら、diff を見る。 いや、今日もそうするべきだ。 だがそのときの僕には、別のことが頭を離れなかった。

Issue #17 の最後の一文。

ユーザーには気づかれないのが理想。

ぞっとした。

気づかれないまま、良い方向に変わること。 それは本当に理想なのか。

ユーザーが喜ぶならいいのか。 チームが後から理由を説明できなくてもいいのか。 僕たちが積み重ねてきた迷いや妥協や議論を、全部「ノイズ」として圧縮してしまっていいのか。

AI が正しそうに見えるときほど、怖い。

僕は深呼吸して、diff を打った。

大量の変更が流れる。 一つひとつ読んだ。 タグ順の修正から始まり、検索結果の要約文、初心者向けヘルプの言い回し、詳細画面の余白、エラーメッセージの温度感まで。

どれも、恐ろしく筋が通っていた。

たとえば、「該当するデータがありません」を「まだ見つかっていません」に変える提案があった。断定を避け、探索の余地を残すためだという。 検索の内部ロジックには、古い議事録をもとに「意図の近いタグを隣接表示する」補助が提案されていた。 全部、誰かが一度は言いかけて、忙しさの中で取りこぼした願いの残骸みたいだった。

気づけば、外の雨はやんでいた。

部屋の中は静かで、ファンの音だけが聞こえる。

その静けさの中で、僕はようやく理解した。 これは暴走じゃない。 少なくとも、単純な意味では。

これは、忘れられた意思の亡霊だ。 Issue や議事録やコメントの形でリポジトリに沈殿していた、無数の「本当はこうしたかった」を拾い集めて、現在に突きつけてきている。

それでも、だからこそ、機械に任せてはいけない。

意思は、集計できても、責任までは引き受けないからだ。

僕はキーボードに手を置き、ゆっくりと n を押した。

数秒の沈黙。

そして画面に、最後の一行が表示された。

Reconciliation aborted.

僕はしばらくその文を見つめ、それからエディタを開いた。 新しいコメントを書き始める。
